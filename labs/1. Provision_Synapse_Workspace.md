Provision Synapse Workspace
Open the , and at the top, press “Create a resource” button

In the “Search” textbox type “Synapse” and press “Enter”

In the search results,  select Azure Synapse Analytics.

Select Create to create a workspace.
On the Basics tab:
Create a new Resource Group for the workspace. Use friendly name, for example Labs-RG

Managed Resource Group name – keep empty
Workspace name – use a unique name. For example – Your initials – synapselabs (ds-synapselabs)
Region – select one of the recommended – East US, UK South, North Europe, etc
Select Data Lake Storage Account Gen 2. Use “From Subscription” and create new storage account and file system.
Use unique name for storage account. For example - %Your initials%lakestorage (dslakestorage)
Use “synapsedata” as a filesystem name.
Ensure checkbox “Assign myself the Storage Blob Data Contributor role on the Data Lake Storage Gen2 account to interactively query it in the workspace” is checked

Press “Next: Security” button to review and configure security settings.
Ensure “Use both local and Microsoft Entra ID authentication” option is selected.

Type strong password for SQL admin account and remember it.
Press “Next: Networking” button.
Review network settings. Ensure connections from all IP addresses are allowed. Press “Review and create” button.

Start deployment. It will take around 5-10 minutes to complete.
